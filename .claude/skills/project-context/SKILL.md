# YumYum 專案背景

## 使用說明

在進行此專案時，請記住以下重點：

1. **專案類型**：這是一個線上多人即時策略堆疊遊戲，靈感來自 Gobblet Gobblers
2. **主要目標**：
   - 展示全端開發能力
   - 用於遊樂、聚會
   - 學習現代技術（WebSocket、Redis、Docker、PostgreSQL）
3. **預算限制**：目標 $0-5/月 + $15/年域名
4. **時程**：四階段開發
5. **設計系統**：黑白配色方案，圓形棋子

## 專案概述

**YumYum** 是一個線上即時對戰的策略堆疊遊戲，靈感來自桌遊 Gobblet Gobblers。

### 核心目標

1. **作品集展示**
   - 展示全端開發能力
   - 微服務架構實作
   - 即時遊戲開發經驗

2. **聚會遊戲**
   - 適合派對場景
   - 手機友善介面
   - 快速上手

3. **技術學習**
   - WebSocket 即時通訊
   - Redis 快取策略
   - Docker 微服務部署
   - PostgreSQL 資料庫設計

### 預算目標

- 開發階段：$0/月
- 正式運行：$0-5/月（在免費額度內）
- 域名：$15/年

### 視覺設計

**配色方案：**
- 主色：純黑 #000000、純白 #FFFFFF、深灰 #333333
- 介面：背景白 #FFFFFF、文字黑 #000000、邊框灰 #CCCCCC、陰影灰 #E5E5E5

**棋子設計：**
- 玩家 1（黑方）：⚫ 小/中/大圓形
- 玩家 2（白方）：⚪ 小/中/大圓形

### 開發時程

- **階段 1**：核心開發（遊戲引擎、基礎 UI、本地對戰、簡單 AI）
- **階段 2**：微服務化（拆分前後端、WebSocket、Redis、Docker）
- **階段 3**：部署上線（Railway、Vercel、域名設定）
- **階段 4**：進階功能（AI 優化、排行榜、成就系統、棋譜回放）

### 成功指標

**技術指標：**
- 遊戲載入時間 < 2 秒
- WebSocket 延遲 < 100ms
- 支援同時 50+ 個房間
- 99% 正常運行時間

**使用者體驗：**
- 規則學習時間 < 5 分鐘
- 手機可正常遊玩
- 無明顯 Bug
- 流暢動畫效果

**作品集價值：**
- 可直接展示給面試官
- 完整的技術文檔
- 開源在 GitHub
- 實際使用者（朋友）反饋

## 範例

**範例 1：檢查設計一致性**
實作 UI 組件時，確保遵循黑白配色方案。主要元素使用 #000000，背景使用 #FFFFFF，邊框使用 #CCCCCC。

**範例 2：預算考量**
在選擇實作方案時，優先選擇能維持在 $0-5/月 預算內的方案。例如，使用 Upstash Redis（免費額度）而非 Railway Redis（需付費）。

**範例 3：時程意識**
目前是階段 1（核心開發）。專注於遊戲引擎和基礎 UI。不要實作進階功能如成就系統或錦標賽 - 那些是階段 4 的內容。
